# PagePurifier

作者：guliacer

一个面向 Tampermonkey / 暴力猴的通用网页净化用户脚本。脚本以保守规则为主，清理常见广告容器、推广卡片、登录营销弹窗、悬浮广告、广告 iframe 和动态插入广告，并集成 B 站 UP 快捷拉黑等个人效率增强，同时支持全局和站点级开关。

## 安装

1. 安装 Tampermonkey 或暴力猴。
2. 打开脚本文件：
   `PagePurifier.user.js`
3. 将脚本内容复制到用户脚本管理器中新建脚本并保存。
4. 刷新网页后生效。

GitHub Raw 安装地址：

```text
https://raw.githubusercontent.com/guliacer/PagePurifier/main/PagePurifier.user.js
```

## 主要功能

- 隐藏常见广告容器、赞助位、推广模块和悬浮广告。
- 移除常见广告网络的 iframe、script、embed 和 object 节点。
- 监听页面后续动态插入的广告，自动二次清理。
- 清理视频站播放器广告层，并尽量恢复播放器控制条。
- 支持 Tampermonkey 菜单开关：
  - 开启/关闭全局去广告
  - 开启/关闭当前网站去广告
  - 开启/关闭 B 站 UP 快捷拉黑
  - 立即重新清理广告

## 设计原则

- 只做广告、推广、遮挡清理和个人账号内的效率操作，不做 VIP 解析、付费内容绕过或接口滥用。
- 站点规则尽量使用“文本信号 + 位置/尺寸约束”，减少误伤正常功能。
- 对高风险站点使用专用规则，避免把过宽选择器扩展到全站。
- 不依赖外部 AI/API，不上传用户页面数据。

## 自定义规则

如果某个网站还有残留广告，可以在 `siteRules` 中增加站点规则：

```js
{
  host: /(^|\.)example\.com$/,
  hide: ['.ad-area', '#sponsor-box'],
  remove: ['iframe[src*="example-ad.com"]'],
}
```

`hide` 适合普通广告容器，`remove` 适合广告 iframe、script 等外部资源节点。复杂弹窗建议新增专用清理函数，优先结合文案、位置和尺寸判断。
